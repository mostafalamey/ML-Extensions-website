# ML Extensions Project Memory Optimization

This document helps GitHub Copilot build optimal memory patterns for the ML Extensions website project.

## Project Overview for Copilot Memory

### Core Business Logic
- **Product**: SketchUp extensions for architects and designers
- **Primary Extensions**: ML Kitchens, ML Doors (with more planned)
- **Business Model**: One-time purchases through Gumroad
- **Target Users**: Architects, interior designers, contractors using SketchUp

### Technical Architecture 
- **Frontend**: React 18+ with Vite 5+ for build tooling
- **Styling**: Tailwind CSS with custom design system
- **E-commerce**: Gumroad integration for payments and digital delivery
- **Performance**: Optimized for Core Web Vitals and SEO
- **Testing**: Playwright for end-to-end testing
- **Deployment**: Static site hosting (target: Vercel/Netlify)

### Code Organization Patterns
```
src/
├── components/         # Reusable UI components
│   ├── shared/        # Generic components (Button, Modal, etc.)
│   ├── product/       # Extension-specific components
│   └── ecommerce/     # Gumroad integration components
├── pages/             # Route-level components
├── data/              # Extension data and configurations
├── hooks/             # Custom React hooks
├── utils/             # Helper functions
└── styles/            # CSS and Tailwind configurations
```

### Naming Conventions
- **Components**: PascalCase (e.g., `ProductCard`, `GumroadButton`)
- **Files**: kebab-case (e.g., `product-gallery.jsx`, `purchase-flow.jsx`)
- **CSS Classes**: Tailwind utilities + custom classes prefixed with `ml-`
- **Props**: camelCase with descriptive names
- **Constants**: UPPER_SNAKE_CASE

## Content Strategy for Memory

### Marketing Copy Patterns
- **Headlines**: Action verbs + specific benefits + time saved
- **Features**: Capability → Benefit → Value proposition
- **CTAs**: Urgency + value + risk reduction
- **Target Keywords**: "SketchUp extension", "kitchen design", "architectural tools"

### User Experience Patterns
- **Navigation**: Logo → Products → Individual Extensions → Support
- **Product Pages**: Hero → Features → Gallery → Pricing → Installation
- **Purchase Flow**: Product page → Gumroad overlay → Thank you → Support
- **Mobile-First**: All components responsive with touch-friendly interactions

### Performance Optimization Patterns
- **Images**: WebP/AVIF with fallbacks, lazy loading below fold
- **Code Splitting**: Route-based + heavy component splitting
- **Caching**: Aggressive caching for static assets, 1-year for images
- **Bundle Size**: <150KB JS, <50KB CSS per route
